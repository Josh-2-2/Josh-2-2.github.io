What I've Built

Wrist Motion Monitor API
⌥ GitHub ↗ API Spec

A production-grade backend for capturing and streaming wrist motion data from dual BNO055 IMU sensors worn during athletic activity. Originally a senior design project, fully rebuilt with modern Python tooling. Includes a Python BLE bridge that connects to the ESP32 hardware and streams sensor readings to the API in real time.

  • Real-time ingestion via WebSocket streaming and a Kafka producer — each IMU reading is persisted to PostgreSQL and published to a topic simultaneously.
  • BLE bridge built with bleak subscribes to GATT notify characteristics on the ESP32, assembles readings from separate quaternion, accelerometer, and gyroscope characteristics, and forwards complete samples over WebSocket.
  • JWT authentication with bcrypt password hashing securing all REST endpoints — modeled after the RFC-compliant OAuth provider built professionally at HBK.
  • 32 automated tests using pytest and an in-memory SQLite database — covers auth, full CRUD, batch upload, duration calculation, and ordering guarantees.
  • Containerized with Docker Compose orchestrating the API, PostgreSQL, Kafka, and Zookeeper — one command to run the full stack.
